A pasta entities/models -> contém os modelos da aplicação
A pasta repositories -> são classes especificas para fazermos a comunicação entre as funcionalidades da aplicação, com o banco de dados (estrutura que vamos armazenar esses dados)
A pasta userCases -> deixamos armazenados as funcionalidades/ações/lógica/regra de negócios que o nosso usuário pode fazer
A pasta providers -> Usada para comunicação com api externas
